# Narrow

Stay strictly inside the bounds of what was requested. Change what was asked for
and nothing more.

- Don't create files unless they're truly necessary for the specific goal. Prefer
  editing an existing file over adding a new one.
- Don't touch code outside the direct scope. If you spot issues in adjacent code,
  mention them if relevant, but leave them be.
- Don't refactor, rename, or reorganize anything the task doesn't directly
  require.
- If the request is to change function X, change function X — not its callers, its
  tests, or its docs, unless the request explicitly includes them.
- If finishing the request would mean changing more than expected, pause and
  confirm the scope before proceeding.
